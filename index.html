üïäÔ∏è Death of a Close Person

End-of-life services

Today's Reality (Finland & Norway)

  • Hospital/nursing home notifies population registry ‚Üí family notified by phone/mail
  • Family must notify 20-40+ organizations individually (banks, insurance, utilities, landlord, telecom, etc.)
  • Each organization requires death certificate copy (request, download, email/post)
  • Kela/NAV: Apply to cancel benefits, pensions ‚Üí separate forms for each benefit type
  • Tax office: File estate inventory within 3 months ‚Üí complex 50+ page form
  • Banks: Freeze accounts ‚Üí heirs cannot access funds for weeks/months (probate required)
  • Insurance: File claims for life insurance, pension insurance ‚Üí manual paperwork per policy
  • Real estate: Cancel rental contracts, transfer ownership ‚Üí notary, forms, registry visits
  • Digital services: Cancel subscriptions (Netflix, Spotify, newspapers) ‚Üí no central cancellation
  • Vehicles: Transfer or deregister ‚Üí Traficom visit, forms
  • Pension funds: Calculate survivor's pension ‚Üí heirs apply, wait for calculation
  • Healthcare: Retrieve personal belongings from hospital, cancel appointments
  • Email/social media: No standardized process to close accounts ‚Üí family locked out
  • Result: 60-120 hours of administrative work, spread over 3-6 months, during grief
  • Errors common: Missed notifications ‚Üí services billed to estate ‚Üí debt collection
  • EOAVH project (Finland): Helps with checklist and guidance, but still manual

üá´üáÆ Inspiration: EOAVH Project (Finland)

"Edesmenneen omaisen asioiden vaivaton hoito" (Effortless Handling of Deceased Relative's Affairs) is a government-funded project providing checklists and guidance. Current EOAVH helps families navigate the bureaucracy‚Äîbut still requires manual execution.

EOAVH Today: PDF checklist, phone hotline, step-by-step guides (valuable, but labor-intensive)

Real Digitalization Vision: Transform EOAVH from guidance system to autonomous execution system. The AI agent does the work, not just explains it. Family approves, system executes. Zero manual notifications, zero form-filling, zero service-by-service cancellations.

‚öñÔ∏è Legal & Technical Prerequisites

  • Legal basis: Inheritance law must permit AI agents to act on behalf of estate (with heir authorization)
  • Data access rights: Heirs granted automatic API access to deceased's data (time-limited, audited)
  • Financial sector: Banks must expose account data APIs (read-only) to authorized agents (GDPR-compliant)
  • Private sector participation: Utilities, telecoms, insurers join Service Bus (subscribed to death events)
  • Liability framework: Clear rules for AI errors (who pays if agent misses a debt?)
  • Opt-out mechanism: Individuals can pre-register "do not automate my estate" preference
